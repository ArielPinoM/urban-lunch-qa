# BUG-003
**Title:** Missing transit ETA on restaurant map pins.

**Component:** Real-Time Tracking MAP UI Component

**Prerequisites**: The user has successfully submitted an order involving dishes prepared by at least one restaurant and is currently on the Real-Time Order Tracking screen.

**Steps to Reproduce**
1. Navigate to the Order Tracking screen.
2. Inspect the active restaurant pins rendered on the map layer.
3. Observe the area immediately above or attached to the restaurant pin to locate the data overlay card.

**Expected Result**: A data overlay component must display the transit ETA countdown timer attached to the corresponding restaurant pin.

**Actual Result**: No data card or overlay component appears over the restaurant pin, meaning the transit ETA timer is completely missing from the map view.

**Severity:** 🟠 Major

**Priority:** 🟧 High

**Environment:** Samsung Galaxy S23+ (Android 16, One UI 8.0)

**Traceability:** Linked to [TC-OT-006](../test-artifacts/test-suites/04_order_tracking.md) / [REQ-OT-002](../docs/requirements/04_order_tracking_specs.md#requirements-matrix)
